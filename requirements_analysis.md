## 1. 引言
本文档在原有需求基础上，引入 **“Agentic Shell”** 核心设计思想。系统不再依赖死板的固定接口，而是通过赋予后端 Agent 强大的本地 Shell 工具链调用权限，实现自适应练习、智能搜题、多媒体对话的高动态闭环。

## 2. Agent 驱动的混合架构分析
系统分为三个核心层级：
*   **Knowledge Repository (静态库)**: 存储于本地磁盘的 Markdown 结构化题目集（由 Python 管道产出）。
*   **Agent Control Center (大脑)**: 
    *   **Logic Engine**: 处理 3/3 过关等核心算法。
    *   **Shell Executor**: Agent 通过执行 `rg`, `sed`, `grep` 等指令直接操作题目库。
*   **Visual Interface (感官)**: Tauri 驱动的图形化界面，负责展现知识图谱与沉浸式练习环境。

## 3. 核心功能模块拆解 (精髓版)

### 3.1 自动化数据生产线 (Python Ingestion)
- **任务**: `Crawler -> VLM Parser -> Metadata Injector`。
- **解析要求**: 在 Markdown 头部注入 JSON 格式的元数据（知识点、难度、步骤数）。

### 3.2 检索 Agent 与工具化 Shell (Tool-Use)
- **精髓**: Agent 不查询 DB 获取题目，而是生成搜索指令。
- **能力**: 
    - **Search**: 利用 `ripgrep` 实现毫秒级的跨文件题目查找。
    - **Extract**: 利用 `sed/awk` 精准提取 Markdown 题目块，最大限度节省 Token。

### 3.3 自适应评估引擎 (The State Machine)
- **任务**: 动态决定推题难度。
- **逻辑**: Agent 分析用户的近期练习流水（Markdown 日志），结合 3/3 规则和 2/3 补偿规则，实时生成下一步学习建议。

### 3.4 智能伴学与多媒体溯源
- **交互**: 支持语音、视频、截图的多轮对话。
- **追问逻辑**: Agent 读取题目上下文和历史录音/视频记录，调用 Gemini 3.1 Pro 提供启发式讲解。

## 4. 数据流与状态机分析
1.  **用户点击图谱**: 触发图谱节点事件。
2.  **Agent 推理**: Agent 运行 `rg` 查找对应难度的 3 道新题。
3.  **动态渲染**: NestJS 下发 Markdown，Tauri 进行富文本渲染。
4.  **实时评估**: 答题后，Agent 判定过关状态，更新 PostgreSQL 中的元数据。

## 5. 数据模型初步设计
- **`Questions_Index`**: 索引 Markdown 文件路径。
- **`Knowledge_Graph`**: 存储由 AI 自动推导的知识点前置依赖（有向无环图）。
- **`User_State`**: 存储用户掌握度 Level。
- **`Multimedia_Logs`**: 存储对话历史的音视频文件映射。

## 6. 技术可行性与风险评估
- **一致性风险**: 不同模型定级标准可能不一。采用“三轮投票定级法”缓解。
- **无效题风险**: 在 Python 管道中增加“AI 模拟解题”步骤，自动剔除逻辑不通或描述不清的题目。
- **性能**: 在局域网环境下，`ripgrep` 的搜索速度完全能支撑万级并发。
