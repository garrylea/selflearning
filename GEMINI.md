# 项目开发准则

1. **逻辑复用**：代码中的函数要尽量复用，相同的逻辑不要写成多个函数。
2. **增量修改**：对于文档或代码的修改都要使用增量修改，不要全量替换。如果增量修改不成功的话，可以要求人工干预。多使用 `diff` 命令。
3. **测试驱动 (TDD)**：采用 TDD (Test-Driven Development) 驱动程序开发。
4. **文件行数限制**：每个程序文件不要超过 1000 行。超过 1000 行的代码应考虑封装成多个功能模块或函数文件进行加载。
5. **分步骤开发**：按照设计文档、接口文档、前端页面原型、开发计划文档进行有步骤的开发。
6. **及时确认需求**：当需求不清楚或存在歧义时，必须及时向用户提问并确认，不可盲目推测。
7. **受控提交**：只有在用户明确要求提交代码（如“提交代码”、“Push”等指令）时才执行 Git 提交和推送操作，不要在每次修改后自动提交。
8. **输出完整性**：大模型输出的内容中，如果与之前内容一致时，不要使用省略号（如 `// ... rest of code`），而是将一样的内容也完整输出出来，确保代码和文档的连续性。
