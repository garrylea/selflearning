# 初中专项自适应学习平台 - 研发前期实施计划 (Pre-development Plan)

本阶段旨在扫清核心技术障碍，并完成系统的全局设计与接口契约制定。遵循“技术验证 -> 架构设计 -> 接口定义 -> 评审”的标准软件工程流程。

## 阶段一：核心技术难点验证与调研 (PoC & Research)
*目标：确定最优的 AI 模型选型与数据预处理 Pipeline。*

*   **任务 1.1：PDF 转 Markdown 模型能力对比测试**
    *   **行动**：编写测试脚本，选取包含“复杂几何图形”和“数学公式”的典型初中真题 PDF 作为样本。
    *   **测试对象**：横向对比 Gemini 1.5 Pro、Gemini 2.5 Pro、Gemini 3.1 Pro、Kimi 2.5、Qwen-VL 等模型。
    *   **输出**：《多模态大模型 PDF 识别能力对比调研报告.md》（评估指标：公式识别准确率、图形还原度/描述能力、生成速度、Token 成本）。
*   **任务 1.2：Tauri 多媒体能力验证**
    *   **行动**：验证 Tauri 容器下 Web Audio API (录音) 和本地文件读写（用于音视频暂存上传）的权限与兼容性。
    *   **输出**：验证结论汇总（作为技术选型文档的支撑）。

## 阶段二：技术选型与系统架构设计 (Architecture Design)
*目标：确立系统的技术栈、服务边界与部署拓扑。*

*   **任务 2.1：确立各模块技术栈**
    *   爬虫与数据管道层：Python
    *   业务后端：Node.js + TypeScript (NestJS / Express)
    *   前端应用：React/Vue + Tauri
    *   持久层：PostgreSQL (关系数据) + Redis (状态缓存) + OSS (多媒体)
*   **任务 2.2：架构图与数据流设计**
    *   输出系统分层架构设计（呈现 Python 数据管道与 Node.js 业务后端的交互方式）。
*   **输出**：《系统架构与技术选型设计说明书.md》

## 阶段三：前后端接口设计与协议制定 (API Design)
*目标：产出可供前后端并行开发的契约文档。*

*   **任务 3.1：RESTful API 核心业务接口设计**
    *   涵盖：知识图谱加载、题目获取、答案提交批改、用户状态同步等。
*   **任务 3.2：WebSocket / SSE 实时交互协议设计**
    *   涵盖：多媒体 AI 伴学对话的流式推送协议、上下文注入格式。
*   **输出**：《前后端 API 接口设计说明书.md》（Markdown 格式，含请求/响应示例体）。

## 阶段四：文档评审与架构冻结 (Review & Freeze)
*目标：确认前置设计，准备进入实质性编码。*

*   **任务 4.1**：对上述产出的《调研报告》、《架构设计说明书》、《API 设计说明书》进行人工评审。
*   **任务 4.2**：修正文档，冻结需求与接口，正式进入工程编码与基础设施搭建。
